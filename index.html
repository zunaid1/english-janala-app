Frequently Asked Questions

What is the difference between var, let, and const ?
var is function-scoped and allows redeclaration, while let and const are block-scoped. let allows reassignment, but const does not. var is hoisted with undefined, whereas let and const are hoisted but stay in a "temporal dead zone" until initialized.
What is the difference between map(), forEach(), and filter() ?
The map(), forEach(), and filter() methods in JavaScript are used for iterating over arrays but serve different purposes. map() creates a new array by applying a transformation function to each element, making it useful when you need a modified version of the original array. forEach() simply executes a function on each element without returning a new array, making it ideal for performing side effects like logging or updating variables. filter(), on the other hand, returns a new array containing only the elements that meet a specified condition. While map() and filter() return new arrays, forEach() does not return anything, making it best suited for actions that don’t require data transformation.
explain arrow functions and how they are different from regular functions
Arrow functions in JavaScript provide a concise syntax using => and differ from regular functions by not having their own this, arguments, or prototype. They inherit this from the surrounding scope, making them useful for callbacks but unsuitable for object methods requiring this. They also cannot be used as constructors and allow implicit returns for single expressions.
How JavaScript Promises work?
JavaScript Promises represent the eventual completion (or failure) of an asynchronous operation and its resulting value. A Promise is in one of three states: pending, fulfilled, or rejected. When a Promise is created, it starts in the pending state and eventually transitions to either fulfilled (with a resolved value) or rejected (with an error). You can handle these outcomes using .then() for fulfillment and .catch() for errors. Promises allow for cleaner, more manageable code compared to callbacks, especially when chaining multiple asynchronous operations.
How closures work in JavaScript?
Closures in JavaScript occur when a function retains access to its lexical scope, even after the outer function has finished executing. This means that an inner function can access variables and parameters from its outer function, even after the outer function has returned. Closures allow for private data encapsulation and are often used for creating functions with preserved states, such as in factory functions or event handlers. They work because JavaScript functions "remember" the scope in which they were created, not just where they were called. This behavior enables powerful programming patterns like data hiding and partial application.
